# 2026-04-16 工作日志

## SDD 开发流程改造 — Skill 全套创建完成

将 long-task-agent 工程改造为符合部门 IPD/SDD 规范的开发流程，创建了 6 个新 Skill：

### 新增 Skill 列表

| Skill | 路径 | 职责 |
|-------|------|------|
| sdd-router | skills/sdd-router/SKILL.md | 路由器，检测 AR 目录状态，分发到对应阶段 skill |
| sdd-task-requirements | skills/sdd-task-requirements/SKILL.md | 读组件详设建立领域知识 + 需求澄清 + 生成 srs.md + tasks.md |
| sdd-task-design | skills/sdd-task-design/SKILL.md | 基于 srs.md 生成 AR 详细设计 design.md |
| sdd-task-develop | skills/sdd-task-develop/SKILL.md | TDD 方式逐任务实现代码（基于 tasks.md）|
| sdd-task-review | skills/sdd-task-review/SKILL.md | 子 Agent 审查：需求符合性(S)、设计符合性(D)、组件规范(C) |
| sdd-task-st | skills/sdd-task-st/SKILL.md | ST 验收测试 + 生成测试用例 + 归档 AR |

### 关键设计决策

- **AR 目录结构**：`specs/changes/ARxxx-topic/`（进行中）→ `specs/archive/ARxxx-topic/`（归档后）
- **文件约定**：srs.md（需求）、design.md（设计，有部门模板）、tasks.md（任务跟踪，Markdown）、st-cases.md（ST 测试用例+报告）
- **tasks.md 状态机**：pending → in_progress → passing；失败时恢复 in_progress
- **部署目标**：OpenCode 平台（非 WorkBuddy）
- **领域知识准备**：每个 skill 启动时必须重新读取组件详设、AGENTS.md、代码结构

### 工程背景

- 流程对齐 IPD：RR→IR→SR→AR，开发团队只接触 SR 和 AR 级别的需求
- srs.md 是对 AR 需求的简洁描述（无模板），design.md 有部门模板（用户暂未提供）
- AR 编号由用户在需求阶段提供，格式自由（如 AR001、AR-20260416）
- AGENTS.md 是可选的（有则读，无则跳过）
